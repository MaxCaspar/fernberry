# Workflow: Convert To Agent

## Required Reading
1. references/mapping-table.md
2. references/codex-skill-best-practices.md

## Process
1. Extract agent definition
   - Identify roles, goals, tools, and constraints.
   - Note any system-level instructions.

2. Translate to Codex skill
   - Map roles/goals into Objective and Essential Principles.
   - Convert tool usage into workflows and references.

3. Split procedures
   - Separate steps into workflows for clarity.
   - Add references for reusable knowledge.

4. Validate constraints
   - Ensure tool usage is compatible with Codex environment.
   - Add verification steps if needed.

## Success Criteria
- Agent behavior represented as a Codex skill.
- Tool usage and constraints are preserved.
- Workflows and references are coherent and testable.
